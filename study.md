## 1. Buck 变换器核心原理

### 1.1 为什么用 Buck（降压变换器），而不是线性稳压？

这是设计中最根本的选择。两种方案对比：

| 特性 | **线性稳压（如 LM7805）** | **Buck 开关变换器** |
|:---:|:---:|:---:|
| 原理 | 像可调电阻：多余电压通过发热消耗掉 | 像快速开关的水龙头：断续取电，平均后得到低压 |
| 效率 | (Vout/Vin) — 30V→5V 时仅 **16.7%** | 典型 **80%~90%+** |
| 发热 | 大输入输出压差时 → 严重发热 | MOSFET 只有导通和截止两种状态 → 发热极小 |
| 体积 | 需要大散热片 | 需要电感和电容，但总体可以更小 |
| 纹波 | 极低（几 mV） | 需要滤波（但可做到 <75mV） |
| 复杂度 | 简单：只需 3 个引脚 | 较高：需要电感、电容、驱动、控制环路 |

**结论：** 本题输入 28~32V，输出 5~24V，最大压差可达 27V（30V→3V）。如果用线性稳压：
- P = I × Vdrop = 3A × 27V = **81W** → 会烧毁
- 而 Buck 变换器的理论效率 ≈ Vout/Vin，即使 30V→5V 也约 **83%**

所以 **必须用开关电源拓扑**，Buck 是最适合"高压差、降压"场景的拓扑。

### 1.2 Buck 工作原理 — 用"水龙头"比喻理解

> 先看视频：[【12v降到5v，开关电源中的Buck电路是怎么做到的】](https://www.bilibili.com/video/BV1Yb4y147iM)

想象一个场景：

```
               🚰 水桶（输入电源 30V）
                │
          ┌─────┴─────┐
          │  快速开关  │ ◄── MOSFET
          └─────┬─────┘
                │
           ╔════╧════╗
           ║  水轮   ║ ◄── 电感（储存/释放水流能量）
           ╚════╤════╝
                │
          ┌─────┴─────┐
          │  水桶     │ ◄── 输出电容（蓄水池）
          └─────┬─────┘
                │
              🚰 用水（负载）
```

**开关打开时（MOSFET 导通）：** 高压水流通过，推动水轮（电感储能）+ 给蓄水池充水（输出电容充电）+ 直接供应用水。

**开关关闭时（MOSFET 截止）：** 水轮靠惯性继续转动，推动水流（电感续流二极管回路），蓄水池继续放水供应用户。

**两个阶段交替运行，水龙头（负载）得到的就是连续、平滑的较低压水流。** 开关速度（频率）越快、蓄水池（电容）越大，水流越平滑（纹波越小）。

### 1.3 Buck 的三种实物状态分析

这是 **同步 Buck**（两个 MOSFET，不用二极管），本题使用 IR2104 驱动 Q1（高侧）和 Q2（低侧）。

#### 状态 ①：Q1 导通、Q2 关断（充电/储能阶段）

```
  VIN(30V) ──────┐
                  │
          ┌───────┴────── Q1 ON（导通，低电阻）
          │                   
     VS点 ───→ L1 ──→ 到输出电容和负载
          │
          ┌─────── Q2 OFF（截止）
                  │
                 GND
```

- **电流路径：** VIN → Q1（导通）→ L1 → 输出电容 / 负载 → GND
- **电感行为：** 电流线性上升，以磁场形式储存能量
- **输出电压变化：** 电容充电，Vout 上升
- **关键公式：** Vout = Vin × D（D = 占空比，理想情况）

#### 状态 ②：Q1 关断、Q2 导通（续流/释放阶段）

```
  VIN(30V) ──────┐
                  │
          ┌───────┴────── Q1 OFF（截止）
          │                   
     VS点 ───→ L1 ──→ 到输出电容和负载
          │
          ┌─────── Q2 ON（导通，低电阻）
                  │
                 GND
```

- **电流路径：** GND → Q2（导通）← L1 电流（电感极性反转，维持电流方向不变）
- **电感行为：** 磁场衰减，电流线性下降，持续向负载供电
- **输出电压变化：** 电容放电，Vout 略微下降
- **关键：** Q2 在此时替代了普通 Buck 中续流二极管的作用，这就是"同步"的含义

#### 状态 ③：占空比与输出电压

MCU 通过 PWM 控制 Q1 的导通时间比例：

```
          ┌──────┐    ┌──────┐    ┌──────┐
  Q1     │ ON   │    │ ON   │    │ ON   │
  Gate   │      │    │      │    │      │
          ┘      └────┘      └────┘      └───
          
          ┌── Ton ──┐←── Toff ──→│
          │◄──── T = 1/f ──────→│
          
          D = Ton / T = Ton × f
          Vout = Vin × D          （连续导通模式，忽略损耗）
```

**例如：** Vin=30V，想要 Vout=12V → D = 12/30 = **40%**  
若 f=100kHz（PWM 频率），周期 T=10μs → Ton = 4μs

> 本题 MCU 输出 PWM 给 IR2104 的 IN 引脚 → IR2104 自动在两个 MOSFET 间生成互补驱动信号。

### 1.4 同步 Buck vs 非同步（普通）Buck — 为什么选两个 MOSFET？

| 类型 | 结构 | 优缺点 |
|:---:|:---:|:---:|
| **非同步 Buck** | 高侧 MOSFET + 低压侧 **肖特基二极管** | ✅ 简单，无需死区控制；❌ 二极管导通压降大（~0.5V），大电流时效率低 |
| **同步 Buck** | 高侧 MOS + 低侧 **MOSFET** | ✅ 低压降（Rds(on)=0.044Ω × 3A=0.132V 压降），效率更高；❌ 需互补驱动 + 死区时间控制 |

**为什么本题选择同步 Buck：**

1. **效率要求：** 发挥部分要求 ≥75%（@12V/2.5A）。非同步 Buck 在低压大电流输出时，二极管的导通损耗会拖低效率：
   - 二极管损耗 = Vf × I × (1-D) = 0.5V × 2.5A × 0.6 ≈ **0.75W**
   - 同步 MOSFET 损耗 = I² × Rds(on) × (1-D) = 2.5² × 0.044 × 0.6 ≈ **0.165W**
   - 仅此一项就节省了 **0.585W**

2. **输出低压时占空比小：** 当 Vout=5V（占空比 ~17%），续流时间占 83%，此时低侧器件的导通损耗更加显著，同步优势进一步体现

3. **题目给了 IR2104 驱动器：** 它能自动处理互补驱动和死区，让同步 Buck 的"复杂度"大大降低

### 1.5 电感值为什么选 1mH？

电感是 Buck 变换器中最关键的储能元件。1mH 是通过 **电流纹波要求** 计算出来的：

$$L = \frac{V_{out} \cdot (V_{in.max} - V_{out})}{f_{sw} \cdot \Delta I_L \cdot V_{in.max}}$$

假设：
- Vin.max = 30V
- Vout = 12V（最严格工况通常取中点附近）
- fsw = 100kHz（典型的开关频率）
- ΔIL（纹波电流）取 Iout.max × 0.3 = 3A × 0.3 = **0.9A**（30% 纹波是常见折中）

计算：

$$L = \frac{12 \times (30-12)}{100\times 10^3 \times 0.9 \times 30} = \frac{216}{2.7 \times 10^6} \approx 0.00008H = 80\mu H$$

Wait — 计算出的 80μH 远小于 1mH。这说明 **1mH 留了很大余量**，或者采用了更保守的纹波要求。

实际中，对于本题：

**1mH 的直觉解释：**
- 电感值越大 → 纹波电流越小 → 输出电压纹波越小
- 电感值越大 → 电感体积越大、直流电阻越大
- 1mH @ 5A 的磁环电感是市面上容易买到的规格

如果我们要把纹波电流控制在更小值（例如 0.3A，即 10%）：

$$L = \frac{12 \times (30-12)}{100k \times 0.3 \times 30} = \frac{216}{0.9 \times 10^6} \approx 240\mu H$$

再加上安全余量和成本考虑，1mH 是一个 **保守可靠** 的选择。较大的电感值使电路更稳定、纹波更小，缺点是电感的物理尺寸略大。

> 💡 **实际工程思考：** 从纹波要求反推——基础部分要求 ≤250mVpp，发挥部分 ≤75mVpp，大电感天生有利于降低纹波。用 1mH 甚至 2.2mH 都是完全可行且常见的选择。

## 2. 器件选型深度解析

### 2.1 IR2104PBF — 半桥栅极驱动芯片

#### 为什么需要专门的驱动芯片？

MCU 的 GPIO 输出 3.3V PWM 信号，**不能直接驱动 MOSFET**，原因：

| 问题 | 说明 |
|:---:|------|
| ❌ **电压不够** | N-MOSFET 完全导通需要 Vgs ≥ 10V，3.3V 只能使 MOSFET 处于"线性区"，会严重发热烧毁 |
| ❌ **电流不够** | 栅极有寄生电容（IRF540 的 Ciss ≈ 1700pF），需瞬时大电流充电才能快速开关 |
| ❌ **高侧 MOSFET 浮地** | Q1（高侧）的源极（VS 点）在 0V~30V 之间摆动，需要在 VS 之上的电压才能驱动，普通电路无法实现 |

**IR2104 解决了这三个问题：**
- 提供 **10~20V 的栅极驱动电压**（由 VCC=12V 供电）
- 提供 **峰值驱动电流**（约 0.3A 拉电流/0.5A 灌电流）
- 内置 **自举电路** 处理高侧驱动（通过外部 D1 + C2）

#### IR2104 的内部框图理解

```
                      ┌──────────────────┐
    IN(2) ────────────┤  电平转换/逻辑     │───→ HO(7)    高侧输出
                      │                   │
    SD#(3) ───────────┤  死区时间控制      │───→ LO(5)    低侧输出
                      │                   │
                      │  ┌─────────┐      │
    VB(8) ─────┬──────┤  │ 自举供电 │      │
               │      │  └─────────┘      │
    VS(6) ─────┴──────┤                   │
                      └──────────────────┘
    COM(4) ───── GND
    VCC(1) ───── +12V
```

**核心功能拆解：**

1. **输入逻辑：** IN 引脚接收 MCU 的 PWM（3.3V 逻辑电平即可，IR2104 识别 3.3V 为高电平）
2. **死区时间：** 当 IN 从高→低或低→高切换时，芯片 **内部自动插入死区时间**（约几 100ns），确保 Q1 和 Q2 **不会同时导通**（否则会形成 VIN → Q1 → Q2 → GND 的直通短路，瞬间烧毁）
3. **自举电路：** 见 [2.3 节](#23-自举电路原理为什么需要-d1fr207--c247μf)
4. **SD#（关断引脚）：** 低电平强制 Q1 和 Q2 都关闭，用于过流保护时快速切断输出

### 2.2 IRF540NPBF — 功率 MOSFET

#### 为什么选 IRF540，而非别的 MOSFET？

看数据手册中的关键参数：

| 参数 | IRF540 | 为什么重要 |
|:---:|:---:|:---|
| **Vds（漏源耐压）** | **100V** | 输入最高 32V，开关瞬间 VS 节点有振铃尖峰，需留余量。100V 很安全 |
| **Id（连续电流）** | **33A** | 输出最大 3A，选择 10 倍余量，开关管温升低 |
| **Rds(on)（导通电阻）** | **44mΩ** | 决定导通损耗：P = I² × R = 3² × 0.044 = **0.4W**（每个管@3A）|
| **Vgs(th)（开启阈值）** | **2~4V** | IR2104 输出 12V 驱动 → 完全导通无压力 |
| **Ciss（输入电容）** | **~1700pF** | 栅极充电所需电荷，影响开关损耗和驱动电流需求 |
| **封装/采购** | **TO-220** | 方便加装散热片，且嘉立创有货 (C2566) |

**为什么不选更小的 MOSFET（如 TO-252 封装的 IRFR540）？**
- TO-252 的散热能力较差，3A 持续输出需额外措施
- TO-220 可以轻松装上小型铝散热片，散热效果明显好

**为什么不选更低 Rds(on) 的器件？**
- 更低 Rds(on) 意味着更大的芯片面积/更高的成本
- 44mΩ × 3A → 0.4W 的发热量已经可以接受
- 这个项目的关键限制不在 MOSFET 导通损耗，而在驱动、电感和 PCB 布局

### 2.3 自举电路原理：为什么需要 D1(FR207) + C2(47μF)？

这是初学者的 **最大困惑点**。

**问题：** Q1（高侧 MOSFET）的源极（VS 点）电平在 0V 和 ~30V 之间快速切换。要让 Q1 导通，栅极电压必须 **比源极高 10V**。当 Q1 源极是 30V 时，需要栅极达到 **40V** — 我们只有 12V 辅助电源，怎么办？

**答案：自举电容升压**

```
                     +12V ──→ D1 ──→ C2(+)──→ VB(8) ←── IR2104 内部 → HO(7)
                                         │
                     VS(6) 节点 ────────┴──(C2 的负极)
```

**工作过程：**

**步骤 A — Q2 导通时（低侧导通）：**
- VS 点被 Q2 拉到 GND（约 0V）
- +12V 通过 D1 给 C2 充电：C2 两端电压 ≈ 12V - 0.7V（D1 压降）= **11.3V**
- VB 点电压 = VS + VC2 ≈ **11.3V**

**步骤 B — 切换到 Q1 导通：**
- IR2104 用 VB-VS 之间的电压（约 11.3V）驱动 HO → Q1 栅极
- Q1 导通后，VS 点电压上升至 ~VIN
- 但 C2 的电压不能突变：当 VS 从 0V 跳到 30V，VB 也从 11.3V 跳到 **41.3V**
- D1 的反向偏置（FR207 的耐压 1000V 绰绰有余）阻止 VB 回流到 +12V
- 此时 HO-VS 之间的电压 ≈ 11.3V，Q1 维持导通

**步骤 C — Q1 关断后回到步骤 A**，C2 再次充电

**为什么 C2 选 47μF？**
- C2 需要储存足够的电荷，在 Q1 导通期间持续为栅极供电
- Q1 栅极每次开关需要电荷：Qg(IRF540) ≈ 72nC
- C2 最小容量估算：C2 >> Qg / ΔV（ΔV 是允许的电压跌落）
- 47μF 远远满足要求（比最小值大 100 倍以上），留了充足余量

**为什么 D1 用 FR207（快恢复二极管）？**
- **反偏恢复速度：** 在高速切换（100kHz）时，普通整流管（如 1N4007）反偏恢复太慢，在 C2 反向时会导致瞬态短路
- **FR207 是快恢复管：** 恢复时间 500ns，适合 100kHz 开关
- **FR207 耐压 1000V：** 自举点的最高电压 ~40V，1000V 属于超大余量

### 2.4 栅极电阻 R2/R4 = 10Ω + D2/D3(1N4148)

#### 为什么栅极要串电阻？

```
  IR2104 HO(7) ───┬── R2(10Ω) ──┬── Q1 栅极
                   │             │
                   └── D2 1N4148 ┤├── Q1 栅极
```

这个配置称为 **"关断优先"栅极驱动**：

**R2=10Ω（开通限流）：**
- MOSFET 栅极是电容负载（Ciss ≈ 1700pF）
- 如果没有电阻，IR2104 的 0.3A 峰值电流会对栅极电容快速充放电
- 过快的开关速度产生两个问题：
  1. **振铃：** 高 di/dt 在 PCB 走线寄生电感上产生振荡，造成电磁干扰（EMI）
  2. **过冲：** Vgs 瞬间超过 ±20V 极限值，损坏 MOSFET
- R2=10Ω 限制了栅极充电电流，使 Q1 开通速度略微减缓，振铃大幅减小

**D2(1N4148) 加速关断：**
- 关断时，D2 旁路了 R2 → 栅极电荷通过 D2 快速泄放
- 这样实现了：**慢开通（抑制 EMI）+ 快关断（降低开关损耗）**
- 1N4148 是通用信号开关二极管，开关速度极快（~4ns），绰绰有余

#### 为什么栅极要有下拉电阻 R5/R6=10kΩ？

```
          IR2104
           │
           10Ω ───┬── Q1 栅极
                   │
              R5=10kΩ ←─── 下拉到 VS（对 Q1）或 GND（对 Q2）
                   │
                  GND
```

**作用：** 在 MCU 未初始化、IR2104 未工作时，MOSFET 栅极是 **悬空** 的 → 栅极上微小的静电噪声即可使 Vgs 超过阈值 → MOSFET **意外导通** → 可能造成短路。

10kΩ 下拉电阻确保：
- 上电瞬间栅极电压被拉到地电平
- MOSFET 保持关断状态，直到 MCU 主动输出 PWM
- 对于 Q1，下拉到 VS（而不是 GND）；对于 Q2，下拉到功率 GND

### 2.5 输入/输出滤波电容 C5/C6=1000μF + C7/C8=100nF

#### 为什么同时并联 1000μF 电解 + 100nF MLCC？

这是一个 **经典的多级滤波** 组合：

```
  C5=1000μF/50V         C7=100nF/50V
  铝电解电容               MLCC（多层陶瓷）
  ──┤├──                   ──┤├──
     │                        │
  低频滤波主力             高频滤波补充
```

| 对比 | 铝电解 1000μF | MLCC 100nF |
|:---:|:---:|:---:|
| **容量** | 大 → 存储电荷多 | 小 → 响应快 |
| **ESR（等效串联电阻）** | 较高（几 Ω ~ 几百 mΩ） | 极低（几十 mΩ） |
| **ESL（等效串联电感）** | 较高（几 nH ~ 几十 nH） | 极低（<1nH） |
| **频率特性** | 低频好，>100kHz 时 ESL 使电容呈感性 | 高频好，可达数 MHz 以上 |
| **作用** | 平滑主纹波、抗负载突变（提供电荷） | 滤除开关噪声尖峰 |

**简单理解：**
- 1000μF 像一个大水桶 → 抑制低频纹波、应对负载突然大电流需求
- 100nF 像一个小水池，响应极快 → 吸收开关管通断瞬间的高频尖峰

**两个电容各管一段频率，缺一不可。**

> 为什么 C5/C6 选 50V 耐压？输入端最高 32V + 开关尖峰，50V 是经济实用的安全裕量。

### 2.6 输出分压电阻 R8(10k) + R9(1k)：为什么分压比是 11:1？

#### 分压设计的核心逻辑

```
  Vout(5~24V) ── R8(10kΩ) ──┬──▶ ADC 引脚 (PA0)
                             │
                          R9(1kΩ)
                             │
                            GND
```

**STM32 的 ADC 输入范围是 0~3.3V**。输出最高 24V（发挥部分达 25V），不能直接接到 ADC。

分压比 = R9 / (R8 + R9) = 1k / (10k + 1k) = **1/11**

所以：
- Vout = 24V → ADC 输入 = 24/11 ≈ **2.18V** ✅
- Vout = 25V → ADC 输入 = 25/11 ≈ **2.27V** ✅（< 3.3V）
- Vout = 30V（极端）→ ADC 输入 = 30/11 ≈ **2.72V** ✅（仍有安全余量）

**为什么选择这个具体比例？**

| 因素 | 计算 | 结论 |
|:---:|:---|:---:|
| 最大输出电压（发挥部分） | 25V | ADC 需 < 3.3V → 分压比 < 3.3/25 = **1/7.58** |
| 保证满量程利用率 | ADC 参考 3.3V，25V→2.27V，只用了 69% | 若用 1/7.5比例，25V→3.3V 刚好满量程 |
| 取整好买的电阻 | 10kΩ / 1kΩ → 比例 1/11 | 比 7.5 更保守、更安全 |
| 余量考虑 | 万一输出有偏离或浪涌，ADC 不容易烧毁 | **1/11 很保守，适合学习阶段** |

**实际软件中如何还原真实电压：**

```
ADC 读数值（0~4095）→ 电压 = ADC值 / 4095 × 3.3V × 11
```

> 💡 **进阶：** 如果你想让 ADC 动态范围利用更充分，可以把 R8 换成 **6.8kΩ**（比例 ≈ 1/7.8），25V→3.2V，几乎满量程。但 1k/10k 是最易买的组合。

#### 为什么电阻要求 1% 精度？

输出电压的检测精度直接影响稳压精度：
- 如果 R8 是 ±1%（10kΩ ± 100Ω），R9 也是 ±1%（1kΩ ± 10Ω）
- 最差情况下两个电阻的误差是 **同向的**（都偏高或都偏低），分压比误差约 ±2%
- 对应的检测误差：12V 测量值可能偏差 ±0.24V
- 这仍在题目可接受范围内

若用 ±5% 电阻，误差可达 ±10%，无法满足发挥部分 0.05V 的显示精度。

### 2.7 电流采样：R10=0.1Ω/2W + 运放

#### 为什么采样电阻取 0.1Ω？

电流采样本质是 **利用微小电阻的压降来间接测量电流**：

| 阻值 | 3A 时压降 | 功耗 (P=I²R) | 优缺点 |
|:---:|:---:|:---:|:---|
| **0.01Ω** | 0.03V | **0.09W** | 压降小，需运放增益高，信噪比差 |
| **0.1Ω** | **0.3V** | **0.9W** ✅ | 压降适中，信噪比好 |
| **0.5Ω** | 1.5V | 4.5W ❌ | 压降太大，功率损耗严重，发热 |

0.1Ω 是 **最佳折中**：
- 3A 时产生 0.3V 压降 → 电压足够大，不易被噪声淹没
- 0.9W 功耗 → 2W 额定功率的电阻，一倍以上安全余量
- 运放增益 10 倍 → 0.3V → 3V，接近 ADC 的 3.3V 满量程

#### 为什么用 2512 封装或康铜丝？

```
  ┌── 0.1Ω / 2W ──┐
  │                │
  │  2512 封装     │  = 6.4mm × 3.2mm 贴片
  │  康铜丝        │  = 小段电阻丝
  └────────────────┘
```

- 普通 0805 电阻额定功率仅 0.125W ← **不可用！**（0.9W >> 0.125W，会烧毁）
- 2512 封装额定功率 1~2W ← 可用
- 康铜丝的温度系数比普通电阻低 → 大电流下阻值更稳定 ← **更优**

#### LM358 差分放大电路深度解析

```
                    Rf2(10kΩ)
                ┌──────────────┐
                │              │
  R10高端 ── Rf1(1kΩ) ──┬── LM358(-) ──┴── U3.1_OUT ──▶ ADC(PA1)
                         │
  R10低端 ── Rg1(1kΩ) ──┴── LM358(+) ──┬── Rg2(10kΩ) ── GND
                                        │
                                       GND
```

**增益计算：** 

$$Gain = \frac{Rf2}{Rf1} = \frac{10kΩ}{1kΩ} = 10$$

**为什么增益选 10？**

- 最大电流 3A → 采样电压 0.3V → 放大 10 倍 = **3.0V**
- ADC 满量程 3.3V，3.0V 留有 9% 裕量（防止尖峰超过 ADC 限幅）
- 在 I=0.1A（发挥部分过载保护下限）→ 0.01V × 10 = 0.1V → ADC 仍可分辨

**为什么不直接买 INA199A1（固定增益 50V/V）？**
- 如 readme 所述：增益 50 倍 → 3A×0.1Ω×50 = **15V >> 3.3V** ❌
- 解决方案（二选一）：
  1. 用 LM358 自己搭差分放大（更灵活，学习价值高）
  2. INA199A1 输出后加一级 1/5 分压（增加复杂度）
- 本设计选择 LM358 ✅

**LM358 的优点：**
- 价格便宜（¥0.3~0.5/个）
- 单电源供电（3.3V 即可工作，虽不是轨到轨，但输出能到 3V 以上）
- 一个封装内含两个运放（另一个可留作备用）

### 2.8 AMS1117-3.3：为什么从 +12V 降压到 +3.3V 要用 LDO？

```
  +12V ──┬── U4(AMS1117-3.3) ──┬── +3.3V
         │   IN(3)   OUT(2)    │
         │          GND        │
        C1(47μF)             C11(100nF)
         │                     │
        GND                   GND
```

**LDO（Low Dropout Regulator，低压差线性稳压器）的选用原因：**

| 对比项 | 用 LDO | 用 DC-DC 转换 12V→3.3V |
|:---:|:---:|:---:|
| 电路复杂度 | ✅ 极简单（3 引脚 + 2 电容） | ❌ 需要电感 + 电容 + 反馈 |
| 纹波噪声 | ✅ 极低（微伏级），MCU 供电需要 | ❌ DC-DC 有开关噪声 |
| 效率 | ❌ (3.3/12)=27.5%，MCU 电流小所以无所谓 | ✅ 80%+ |
| 成本 | ✅ ¥0.5~1.0 | ❌ ¥2~5 + 外围 |

**核心逻辑：** MCU 电流约 50~100mA，功耗为 (12V-3.3V)×0.1A = 0.87W，AMS1117-3.3 的 SOT-223 封装可以承受。如果用 DC-DC 省下 0.6W 但增加了体积、成本、噪声，得不偿失。**对于小电流降稳压，LDO 是正确选择。**

### 2.9 I2C 上拉电阻 R11/R12=4.7kΩ

```
  PB6(SCL) ───┬── R11=4.7kΩ ─── +3.3V
               │
              OLED SCL ─── (同一条线)

  PB7(SDA) ───┬── R12=4.7kΩ ─── +3.3V
               │
              OLED SDA ─── (同一条线)
```

**为什么 I2C 需要上拉电阻？**
- I2C 是 **开漏（open-drain）** 总线：器件只能将总线拉低（输出 0），**不能** 主动拉高（输出 1）
- 所以需要电阻将总线"默认"拉到高电平
- 当没有器件拉低时，总线为高（逻辑 1）；有器件拉低时，总线为低（逻辑 0）

**为什么选 4.7kΩ？**

上拉电阻的选择是权衡：

| 电阻值 | 过小 | **4.7kΩ** ✅ | 过大 |
|:---:|:---:|:---:|:---:|
| 功耗 | 大（I=V/R=3.3/1k=3.3mA） | **0.7mA**（合理） | 小 |
| 信号上升速度 | 快（RC 时间常数小） | 标准模式 100kHz 够用 | 慢（400kHz 模式下可能不够） |
| 抗干扰 | 好 | 好 | 差 |

OLED 的 SSD1306 在 I²C 模式下通常工作在 100kHz~400kHz，4.7kΩ 是 SSD1306 数据手册 **推荐的典型值**。

### 2.10 防反接 D4(SS34) + TVS 管

#### SS34 肖特基二极管

```
  VIN ────── D4(SS34) ──┬── 进入 Buck 电路
                         │
                        (后级)
```

- **正向压降低（~0.45V）：** 大电流下发热小
- **额定电流 3A：** 系统最大 3A，留有足够余量
- **40V 耐压：** 输入最高 32V，40V 留有 25% 余量
- **原理：** 如果用户将 VIN 和 GND 接反 → D4 反偏 → 电流不通 → 保护后级电路

#### TVS 管（瞬态电压抑制）

```
  VIN ──┬── P6KE36A ── GND
         │   (双向 TVS)
```

- **正常时：** 高阻抗，不影响电路
- **过压时（>36V）：** 雪崩击穿，将电压钳位在安全范围
- **作用：** 防止输入电源浪涌或接触静电时的高压损坏电路

## 3. 辅助电源深度解析

### 3.1 为什么需要 +12V 辅助电源？

回顾电路中需要 +12V 供电的器件：

| 器件 | 供电引脚 | 电压需求 | 原因 |
|:---:|:---|:---:|:---|
| IR2104 | VCC(1脚) | **10~20V** | 栅极驱动电压必须足（见 2.1 节） |
| 自举电路 | VB(8脚) | 依赖 +12V | D1+C2 从 +12V 产生自举电压 |
| 蜂鸣器（可选） | BZ1 | 3.3V/5V | 用 3.3V 从 MCU 板取 |
| AMS1117-3.3 | IN | 输入 4.75~12V | 从 +12V 降压到 +3.3V |

**关键点：** IR2104 的供电电压决定了 MOSFET 的栅极驱动电压（Vgs）。IRF540 的典型开启阈值是 2~4V，但 **完全导通（深线性区）需要 Vgs ≥ 10V**。

```
  Vgs = 4.5V  → Rds(on) ≈ 100mΩ（未完全导通 → 发热大）
  Vgs = 10V   → Rds(on) ≈ 44mΩ（数据手册标注值）
  Vgs = 12V   → Rds(on) ≈ 40mΩ（最佳状态）
```

所以 +12V 不仅是给芯片"供电"，更是为了保证 MOSFET 的导通效率。如果用 +5V 给 IR2104 供电 → Vgs 只有 5V → Rds(on) 翻倍 → 发热量翻倍 → 效率降低。

### 3.2 为什么大二可以直接买商用模块？

**题目说明：** "辅助电源板块大二同学可购买商用产品"

商用 +12V 输出模块主要有两类：

| 类型 | 代表 | 特点 |
|:---:|:---|:---|
| **AC-DC 模块** | HLK-PM01 | 输入 220V AC → 输出 12V DC，隔离型 |
| **DC-DC 模块** | 任何 12V 适配器 | 更常见：直接用一个 12V 手机充电头改装 |

**不建议自制的理由：** 辅助电源设计涉及反激式变换器、变压器设计、反馈环路补偿等，属于另一门学问。本题的 **核心是 Buck 变换器**，不要在辅助电源上消耗过多时间。

### 4.2 为什么 PWM 需要经过 IR2104 而不是直接驱动 MOSFET？

```
  ❌ 错误思路：
  MCU PA6(3.3V PWM) ──── 直接接 Q1 栅极
  
  ✅ 正确路径：
  MCU PA6(3.3V PWM) ──── IR2104.IN ──── IR2104.HO(12V PWM) ──── Q1 栅极
                                          IR2104.LO(12V PWM) ──── Q2 栅极
```

**核心原因（复习 + 深化）：**

1. **电平转换：** MCU 3.3V → IR2104 输出 ≈ +12V 驱动
2. **互补驱动：** IR2104 自动将 IN 的 PWM 分成两路：HO（跟随 IN）和 LO（反向 IN）
3. **死区插入：** 防止 Q1/Q2 瞬时同时导通（导通瞬间 ≈ 短路，烧毁 MOSFET）
4. **驱动能力：** IR2104 提供 0.3A+ 的驱动电流，MCU GPIO 只有几 mA

**死区时间的重要性：**

```
     时间轴 →                                    
  IN ────┐      ┌──┐      ┌──┐      ┌──┐
          └──────┘  └──────┘  └──────┘  └────
  
  HO ────┐    ┌──┐      ┌──┐      ┌──┐
          └────┘  └──────┘  └──────┘  └──
                            ← 死区 →
  LO ───────┐  ┌──┐      ┌──┐      ┌──┐
             ─┘  └────────┘  └────────┘  └─
```

在 HO 关断后、LO 导通前，有一段 **两者都关断的空白时间** → 这就是死区。

**如果没有死区：** HO 关断 → Q1 关断（但关断需要时间）→ 同时 LO 已经输出高 → Q2 开始导通 → Q1 还没完全关断时就形成了 VIN → Q1 → Q2 → GND 的**直通**，一瞬间大电流烧毁 MOSFET。

### 4.3 VS 节点的关键作用 — 承上启下的"开关点"

在原理图中，VS 节点同时连接着：

```
  Q1 源极 ──── Q2 漏极 ──── L1 输入端 ──── IR2104.VS(6) ──── C2(-)(自举)
```

VS 是整 Buck 电路中 **电压变化最快** 的点：
- Q1 导通时：VS = VIN（0V → 30V 跳变）
- Q2 导通时：VS = GND（~0V）
- 在 100kHz 频率下，VS 在每个周期都在 0V 和 30V 之间切换

**对 PCB 布局的启示：**
- VS 走线必须 **短而粗**（减小寄生电感和电阻）
- VS 节点的铜皮面积不宜过大（减少 EMI 辐射）
- VS 节点是测量 Buck 开关波形的 **最核心测试点**（用示波器探头接地弹簧测 VS 对 GND 波形）

### 4.4 AGND 与 PGND 的分割 — 为什么不能混在一起？

```
        ┌──── 信号地 AGND ────┐
        │                      │
  MCU ──┤                 R19(0Ω) ├── 在这个单点连接
  IR2104 COM ──┤                 │
  分压电阻 ──┤                      │
  运放 ──┤                 ┌──── 功率地 PGND ────┐
        │                      │
        └──────────────────────┘  Q2 源极 ──┤
                                 输入/输出电容 ──┤
                                 输入/输出端子 ──┤
```

**为什么分层？** 这是一个极为关键但也常被初学者忽略的设计：

- **大电流回路（功率地 PGND）：** Buck 回路中流过大电流（3A），在 GND 走线上产生压降
- **小信号回路（信号地 AGND）：** ADC 采样、MCU 控制等微弱信号（μV~mV 级别）

如果两个地完全混在一起：

```
  ❌ 错误布局：
                               Q2 源极的 3A 返回电流
                                   │
                    MCU ──── GND ──┴── 采样电阻 GND
                              ↑
                  功率电流在地线上产生 0.1V 压降，
                  被 ADC 采样电路"看到" → 测量误差
```

**正确做法：** AGND 和 PGND 在 **唯一一点** 通过 0Ω 电阻或磁珠连接。这样功率地的噪声电流全部经过短路径流回电源，不会流经信号地。

       └── ...

### 5.2 PID 控制原理 — 零基础理解

#### 什么是 PID？为什么需要它？

**问题场景：** 你设定输出 12V，MCU 通过 ADC 读到实际是 11.5V。你需要调节 PWM 占空比。但：
- 一次调太多 → 输出变成 12.8V（过冲）
- 一次调太少 → 要很久才能到 12V
- 轻载和重载时响应的特性不同
- 输入电压波动时也需要自动补偿

**PID = 比例（P）+ 积分（I）+ 微分（D）**，三个分量各司其职：

#### P（比例）— "看当前误差，乘以系数"

$$输出调整量 = K_p \times (目标值 - 当前值)$$

```
  设定 12V，当前 11.5V → 误差 = 0.5V → 调整量 = Kp × 0.5
  随误差减小，调整量越来越小 → 逐渐逼近目标
  ✅ 优点：响应快
  ❌ 缺点：纯 P 控制会有"静差"（永远差一点到不了目标）
```

#### I（积分）— "看过去误差的累积，消去静差"

$$输出调整量 = K_i \times \int_{0}^{t} 误差(\tau) d\tau$$

```
  如果一直有 0.05V 的误差，积分项会不断累积 → 调整量持续增加 → 直到误差消除
  ✅ 优点：消除静差
  ❌ 缺点：积分过强导致过冲/振荡
```

#### D（微分）— "预测未来的趋势"

$$输出调整量 = K_d \times \frac{d(误差)}{dt}$$

```
  如果电压正在快速下降（负载突然加大），微分项会提前预测到"即将偏离目标" → 提前增加占空比
  ✅ 优点：抑制过冲，提高稳定性
  ❌ 缺点：对噪声敏感（噪声会使"导数"忽大忽小）
```

#### 类比理解：

```
  PID = 你洗完手去关水龙头：
  
  P（比例）→ 看到水流太大，先关一半
  I（积分）→ 关了一半发现还有滴答，再微调一点点直到完全停止
  D（微分）→ 看到水流正在快速变小，"预测"快关到底了，放慢速度以免关过头
```

#### PID 参数整定口诀：

```
  参数    上升时间    过冲    稳定时间    静差    稳定性
  ↑Kp     ↓(变快)     ↑       ↑           ↓       ↓
  ↑Ki     ↓(略快)     ↑       ↑           消除     ↓
  ↑Kd     ↓(略快)     ↓       ↓           —       ↑
```

### 5.3 增量式 PID 代码逐行解析

来自 `buck相关文章.txt` 中的参考代码：

```c
typedef struct PID 
{ 
    int SetPoint;        // 设定目标（期望电压值，如 12.00V 对应 1200）
    long SumError;       // 误差累计（用于位置式 PID，增量式暂时不用）
    double Proportion;   // 比例常数 Kp
    double Integral;     // 积分常数 Ki
    double Derivative;   // 微分常数 Kd 
    int LastError;       // 上一次误差 E[k-1]
    int PrevError;       // 上上次误差 E[k-2]
} PID;
```

**为什么需要存两个历史误差？** 增量式 PID 公式：

$$\Delta U(k) = K_p[E(k) - E(k-1)] + K_iE(k) + K_d[E(k) - 2E(k-1) + E(k-2)]$$

需要用当前的 E(k)、上一次的 E(k-1)、上上次的 E(k-2) 计算。

```c
int IncPIDCalc(int NextPoint) 
{ 
    register int iError, iIncpid;
    
    // 计算当前误差：目标 - 实测
    iError = sptr->SetPoint - NextPoint; 
    
    // 增量式 PID 核心计算
    iIncpid = sptr->Proportion * iError           // P: Kp × E(k)
            - sptr->Integral * sptr->LastError    // I: Ki × E(k-1) [注意这里是减号]
            + sptr->Derivative * sptr->PrevError; // D: Kd × E(k-2)
    
    // 更新历史值
    sptr->PrevError = sptr->LastError;   // 当前的 E(k-1) 变成下一轮的 E(k-2)
    sptr->LastError = iError;             // 当前的 E(k)  变成下一轮的 E(k-1)
    
    // 返回增量值（MCU 将此值加到当前 PWM 占空比上）
    return(iIncpid); 
}
```

**值得注意的地方：**
- 积分项用的是 `- Ki × LastError` 而非 `+ Ki × Error`，这是增量式的一种常见变体写法。相当于将积分作用体现在了误差的变化上而非绝对值上。
- 返回的是 **增量**，不是新的占空比值。所以调用方需要：`PWM_Duty += IncPIDCalc(currentVoltage);`
- PID 的参数（Kp, Ki, Kd）需要通过实验整定，没有万能值。

### 5.4 PID 参数怎么调？（工程实用方法）

#### 方法一：临界比例法（Ziegler-Nichols）

```
  步骤 1：Ki = Kd = 0，只保留 Kp
  步骤 2：缓慢增大 Kp，直到输出电压开始等幅振荡
  步骤 3：记下此时的 Kp = Ku（临界增益），振荡周期 = Tu
  步骤 4：按以下表格设置：
  
           Kp          Ki          Kd
         0.6×Ku    1.2×Ku/Tu   3×Ku×Tu/40
```

#### 方法二：经验试凑法（推荐初学者）

```
  步骤 1：Ki = Kd = 0，Kp 从小开始（如 0.1）
          → 观察输出响应
  
  现象                     → 解决方法
  响应太慢（久久到不了设定值）→ ↑Kp
  响应有振荡但最终稳定       → ↓Kp 或 ↑Kd
  有静差（差一点点到不了）   → ↑Ki（从 0.01 开始试）
  过冲大                   → ↑Kd（从 0.01 开始试）
```

**本题的 PID 调试点：**
- 先空载调 Kp（找到能稳定但不振荡的 Kp）
- 加 10Ω/12V 负载（1.2A）观察电压跌落和恢复
- 再调 Ki 消除静差（带载后电压下降了多少，能否自动调回来）
- 最后调 Kd 抑制过冲（改变设定电压时观察超调量）

## 6. 常见错误与排查

### 6.1 上电后输出为 0V

| 可能原因 | 排查方法 | 解决方案 |
|:---:|:---|:---|
| 🔴 MCU 未输出 PWM | 示波器测 PA6：是否有方波 | 检查程序是否运行、定时器配置 |
| 🔴 SD# 被拉低（关断） | 测 IR2104 的 SD#(3脚) 电压 | 检查 MCU 的 PB0 输出 |
| 🟡 IR2104 无供电 | 测 VCC(1脚) → GND：是否 12V | 检查辅助电源连接 |
| 🟡 自举电容未充电 | 测 VB(8脚) → VS(6脚) 电压：~11V？ | 检查 D1/C2 焊接 |
| 🟡 保险丝熔断 / D4 反接 | 测 VIN 是否到 Buck 电路 | 检查 D4 方向、F1 |

### 6.2 输出电压不准

| 可能原因 | 排查方法 | 解决方案 |
|:---:|:---|:---|
| 🟡 分压电阻装错 | 实测 R8/R9 阻值 | 替换为正确值 |
| 🟡 ADC 参考电压不准 | 测 STM32 的 VREF 引脚 | 确保 3.3V 稳压准确 |
| 🔴 PID 参数未整定 | 用示波器看输出电压是否稳定 | 调整 Kp/Ki/Kd |
| 🔴 PWM 频率不合适 | 测 PA6 的 PWM 频率 | 检查 TIM 配置（建议 50~100kHz） |

### 6.3 输出电压纹波过大

| 可能原因 | 排查方法 | 解决方案 |
|:---:|:---|:---|
| 🔴 输出电容容量不足 | 示波器 AC 耦合测 Vout 纹波 | 确保 C6=1000μF + C8=100nF；可额外并联 10μF MLCC |
| 🟡 电感饱和 | 大电流时听到啸叫？ | 检查 L1 饱和电流是否 ≥5A |
| 🔴 PCB 布局不当 | VS 节点回路面积过大？ | 尽量缩小功率回路面积 |
| 🟡 死区时间不足 | 示波器看 HO/LO 是否有重叠导通 | IR2104 内部死区已处理，检查 PCB 走线寄生参数 |

### 6.4 MOSFET 严重发热

| 可能原因 | 排查方法 | 解决方案 |
|:---:|:---|:---|
| 🔴 栅极驱动电压不足 | 测 Q1 栅极对源极电压：<8V？ | 检查 IR2104 供电是否 12V |
| 🔴 开关频率过低 | 测 PA6 PWM 频率：<20kHz？ | 提高频率到 50~100kHz |
| 🟡 死区时间不足 | 测 HO/LO 是否有瞬间重叠 | 确保 IR2104 供电正常（内部死区依赖 VCC） |
| 🟡 散热不够 | 触摸 TO-220 是否烫手 | 加装散热片 |

### 6.5 开机烧保险丝/冒烟

| 可能原因 | 排查方法 | 解决方案 |
|:---:|:---|:---|
| 🔴 Q1 和 Q2 同时导通（直通） | 不接输入，测 HO 和 LO 对 VS 的波形 | 检查 IR2104 供电、检查 MCU PWM 程序 |
| 🔴 电容接反 | 电解电容有极性标记 | 铝电解反接会爆炸！仔细检查 |
| 🟡 焊锡短路 | 目检 + 万用表蜂鸣档测 VS→VIN/GND | 清理焊点 |
| 🟡 负载短路 | 测 P2 端子两端：短路？ | 检查负载连接 |
